Following are the problem and their solutions which user encounter while changing the username of the Administrator account in a linux machine with their solution. 


Problem 1:
As you are changing the username of Administrator account, you can't change while begin logged into the Adminstrator account. Also for these change you need to have adminstrator privileges which is not present when you logged in as Guest User.

Solution:

We have to enable the Root account of the Linux. This account has the Admin rights so we can perform desired action. To enable Root account,

  1. Create a root password by executing following command sudo passwd root
  2. Enable the Root Login by executing command sudo passwd -u root


After enabling this Root account, we have to logout from current system and login as Root.


Problem 2: [Conditional]

Sometimes even after logging out of the Administrative account, user still remain active. We will not be able to change the username till the user is logged in the system. So we have to deactivate the logged user.

  • Stop the Administrative user by executing command pkill -STOP -u <username>


As we have stopped the Admin user we can change the username by executing usermod -l <new name> <old name>". 
This command will successfully change the username of the given account.


After you have successfully change the username, You should disable the ROOT user by executing command sudo passwd -l root



NOTE:

Above process will only change the username of the account. The home directory will still be named as previous username and also other user based linux configurations.
