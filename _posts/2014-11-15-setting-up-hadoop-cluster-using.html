Note: This post is specific to Digital Ocean cluster but can be used as reference to install Hadoop on other Cloud provider also. You need to have an account on Digital Ocean to perform following steps in the blog.

Step 1: Creating a Server on Digital Ocean.

I will be creating 3 droplets with following name,
  • edgenode.sleepythread.com
  • namenode.sleepythread.com
  • datanode.sleepythread.com
You are free to use any host name you like. Just replace your custom host name instead of above in blog.

1. Size:

First we need to create 3 droplet on Digital Ocean. The configuration which I have used for cluster is following,

Size: 4 CPU/ 8 GB droplet @ 0.119$/hr

2. Region: 

You can choose any region you want but make sure all the nodes are in same cluster. I have selected Singapore Data Center

3. Available Setting: 

Leave all of them blank.

4. Image: 

Ubuntu 14.04 x64 machine


5. SSH keys:

It is advisable to add your public ssh keys in Digital Ocean account, so you need not worry about password. Digital Ocean will automatically enable password less ssh log in. You just need to select your ssh keys.




Step 2: Adding Server information into local host file.

We need to add the server information into our local host file.
## 1. Add entry in /etc/hosts
 
128.XXX.XXX.XXX edgenode.sleepythread.com
 
128.XXX.XXX.XXX namenode.sleepythread.com
 
128.XXX.XXX.XXX datanode.sleepythread.com

Step 3: Download Cloudera Manager on edge node

 

## SSH onto edge node.

ssh root@edgenode.sleepythread.com

## Download latest Cloudera Manager installer

wget http://archive.cloudera.com/cm5/installer/latest/cloudera-manager-installer.bin

## Give execute permission to installer file

chmod u+x cloudera-manager-installer.bin

## Run the installer:
./cloudera-manager-installer.bin


You will be asked to accept Oracle and Cloudera License. After that Installer will install cloudera manager on the edge node.

At the end you will see installation successful message.  Now you can access the manager on 7180 port on edge node.  

Step 4: Setting Up Password, Host Entry and Ports for installation


Cloudera manager requires user name & password to download and install various component onto the machine. Hence we need to setup same password for all the machine's in the cluster. Also to effectively communicate, It will be helpful to add host entry of all machine into each nodes host file. For security reasons, Digital Ocean has iptables enabled on each port. We need to open some port for cloudera manager to install and validate component in cluster. 

## 1. Change Password:
 
passwd
 
## 2. Add following entry in /etc/hosts And Remove the 127.0.0.1 entry for hostname.  
## e.g: 127.0.1.1 edgenode.sleepythread.com edgenode.sleepythread.com
 
 
128.XXX.XXX.XXX edgenode.sleepythread.com
 
128.XXX.XXX.XXX namenode.sleepythread.com
 
128.XXX.XXX.XXX datanode.sleepythread.com
 
## 3. Open port for access using IP tables.
 
sudo iptables -A INPUT -p tcp --dport 7182 -j ACCEPT
 
sudo iptables -A INPUT -p tcp --dport 9000 -j ACCEPT
 
sudo iptables -A INPUT -p tcp --dport 9001 -j ACCEPT
 
sudo iptables -A INPUT -p tcp --dport 8020 -j ACCEPT


Step 4:  Using Cloudera Manger to install Hadoop

1. Log in into cloudera manager from browser & accept the licence.

Log in to cloudera manger using URL "http://edgenode.sleepythread.com:7180/cmf/login". 
Default username and password is "admin"



2.  Specify all machine in the cluster:

 We need to specify all the host on which we need to install cluster.



3.Selecting Repository:


You can now choose the repo and version of CDH which you want to install on your cluster.



4. JDK installtion



 5. Adding SSH login credentials:

We need to add ssh login information to cloudera manager as it will be used to execute installation command to each host in the cluster. Add "root" & password which you have set on Step 2.



Now Cloudera will install its agent on each host.

 After sometime you will get installation success message



After continuing this step, Cloudera will install all CDH package on all the host.


Next will perform Host inspection, you can skip if you want.

6. Select Installation type

 You can now choose which type of installation you want for your cluster. We will be installing Core Hadoop.




7.  Setting location for each Service on the Cluster


You need to select where each service need to be installed on your cluster:

Hue:

ZooKeeper:



8. Setting up Database for Hive & Cloudera Manger

Hive & Cloudera Manger need database to keep metadata and information respectively. 





9. Reviewing Change:

We can use the default configuration.



10. Cluster setup


Now Cloudera will install and start all the components on specified host.




Once manager has installed all the component you can view your cluster


Note: As I have installed data node and node manger on Namenode machine too, you will see Configuration warning as Cloudera manger had configured Namenode will less memory. In real Cluster, Namenode will not have any other service running on same machine.


Congratulation, your cluster is up and running. You can use Hue to interact with your cluster.


Additonal Setting: Oozie Web Console is not yet setup and we need to explicitly as follows,

1. Select Oozie from the panel.
2. On Oozie configuration page select Configurations
3. Search for "console", Check the Value checkbox to enable console.
4. Download and extract ext.js in "/var/lib/oozie" on node which oozie is install.

## SSH on node which oozie is install
 ssh hadoop@edgenode.sleepythread.com

# Install unzip to unzip ext zip.

sudo apt-get install unzip

#  Download ext zip in /var/lib/oozie

cd /var/lib/oozie

wget http://extjs.com/deploy/ext-2.2.zip

unzip ext-2.2.zip
