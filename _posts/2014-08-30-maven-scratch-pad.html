Commit 1:

Creating simple jar using Maven & Intellij

Maven follows convention over configuration philosophy and hence there are a very few details need by Maven to create an executable JAR.

  1. Create a directory maven-tutorial
  2. Create a file pom.xml inside directory.
  3. Add project information
    1. modelVersion : this attribute specify version of the object model this POM is using.
    2. groupId: unique identifier of the organization or group that created the project.
    3. artifactId : This element indicates the unique base name of the primary artifact being generated by this project. 
    4. version: version of the artifact generated by the project.
  4. Open project in Intellij
    1. Choose open project from File menu.
    2. Navigate the directories and choose the pom.xml file.
  5. Create directory src/main/java inside project.
    1. Go To Intellij Project structure > Module section and mark src/main as sources
  6. Create package same what you have given in groupId
  7. Create a Main Class
  8. Add an SOUT command in main function in class.
  9. Go to console and type mvn package
    1. This command will now create a jar in target/ directory in same folder.
  10. Run the Jar using command "java -cp target/maven-tutorial-0.1-SNAPSHOT.jar org.sleepythread.Main" to Run the Jar. 

Commit 2: 

Adding Testing Capability [Specifying compile time dependency.]

As you have seen in the log while doing mvn package in last commit, maven tries to run test. This is due to convention over configuration philosophy where it tries to find test and run it. Time to write some test.
  1. Adding dependencies information in pom.xml. 
  2. Each dependency require following information
    1. groupId: similar to our maven pom file. Info is present from maven website.
    2. artifactId: similar to our maven pom file. Info is present from maven website.
    3. version: version which need to be used. Info is present from maven website.
    4. scope: This specify when is given dependency is required for the project. There can be following values for scope
      1. compile: This is the default scope, this will include maven dependency in your classpath.
      2. provided: specify that given dependency will be available in by running environment like web server e.t.c
      3. runtime: specify that given dependency is not required at compile time but its needed while running the app. It is in the runtime and test classpaths
      4. test: specify that given dependency is required to run test. It will be available in compilation and execution phase. 
      5. system: similar to provided except that you have to provide the JAR which contains it explicitly.
  3. Add Test
    1. create test directory src/test.
    2. Add package org.sleepythread
    3. Add Test file with simple test case.
  4. Re run mvn package command. Now you can see the logs contain. -------------------------------------------------------
     T E S T S
    -------------------------------------------------------
    Running org.sleepythread.MainTest
    Tests run: 1, Failures: 0, Errors: 0, Skipped: 0, Time elapsed: 0.058 sec
